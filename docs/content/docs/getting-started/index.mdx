## What you will accomplish

You will create an Axum server that returns an Inertia page, connect a Vite frontend, and open the result in a browser. The [quick start](/docs/getting-started/quick-start) is designed to reach that result in under ten minutes.

## Prerequisites and features

- Rust 1.88 or newer.
- Node.js 22.12 or newer and pnpm.
- `inertia-axum` 1.0.0-alpha.1.
- The default `macros` and `vite` Cargo features. The first application does not need `ssr` or a custom root template.

## How it works

Your handler returns a page response such as `page!("Home", { greeting: "Hello" })`. `RouterInertiaExt::inertia` installs request parsing and response finalization around the router. An initial browser request receives complete HTML; a later request carrying `X-Inertia: true` receives the page object as JSON.

The frontend resolves the component name `Home` to a file under `frontend/src/Pages`. Axum remains responsible for routing, state, extractors, middleware, validation, and redirects.

## Defaults and edge cases

- `InertiaApp::vite("frontend")` expects `frontend/src/main.ts` and `frontend/dist/.vite/manifest.json` unless you override the entry or build directory.
- The built-in root document is the default. A template file is optional.
- `build()` is synchronous and is correct for CSR-only startup. Configured SSR requires asynchronous `start()`.
- Production startup fails early when the Vite manifest or configured entry is missing. Development can use `VITE_DEV_SERVER_URL` instead.

## Production guidance

Commit the frontend lockfile, run the Vite production build before starting Rust, and build the server with `--release`. Add SSR only after the CSR application works; the same routes continue to serve both modes.
