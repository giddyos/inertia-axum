## What you will accomplish

You will create a new application, generate one frontend page, build its assets, and serve `Hello from inertia-axum` at `http://127.0.0.1:3000`.

## Prerequisites and features

Complete [Installation](/docs/getting-started/installation) first. This guide uses the default `macros` and `vite` features. It does not require `ssr`, a session store, or a custom root document.

## Minimal copyable implementation

<Steps>
  <Step>
    ### Create the Rust project

    ```bash
    cargo new inertia-demo
    cd inertia-demo
    cargo add axum@0.8.9 --features http1,tokio
    cargo add inertia-axum@1.0.0-alpha.1
    cargo add tokio@1 --features macros,net,rt-multi-thread
    cargo install cargo-inertia --version 1.0.0-alpha.1 --locked
    ```
  </Step>
  <Step>
    ### Generate one frontend

    Choose the framework you want to use:

    <Tabs items={['Svelte', 'React', 'Vue']}>
      <Tab value="Svelte">
        ```bash
        cargo inertia init --frontend svelte
        ```
      </Tab>
      <Tab value="React">
        ```bash
        cargo inertia init --frontend react
        ```
      </Tab>
      <Tab value="Vue">
        ```bash
        cargo inertia init --frontend vue
        ```
      </Tab>
    </Tabs>

    The command creates `frontend/package.json`, `frontend/vite.config.ts`, `frontend/src/main.ts`, and `frontend/src/Pages/Home.*`.
  </Step>
  <Step>
    ### Return the `Home` page from Axum

    Replace `src/main.rs` with this reduced version of the maintained `axum-minimal` example:

    ```rust title="src/main.rs"
    use axum::{routing::get, Router};
    use inertia_axum::prelude::*;

    async fn index() -> DynamicPage {
        page!("Home", { greeting: "Hello" })
    }

    #[tokio::main]
    async fn main() -> Result<(), Box<dyn std::error::Error>> {
        let inertia = InertiaApp::vite("frontend").build()?;
        let app = Router::new()
            .route("/", get(index))
            .inertia(inertia);

        let listener = tokio::net::TcpListener::bind("127.0.0.1:3000").await?;
        axum::serve(listener, app).await?;
        Ok(())
    }
    ```
  </Step>
  <Step>
    ### Build and run

    ```bash
    pnpm --dir frontend install
    pnpm --dir frontend build
    cargo run
    ```

    Open `http://127.0.0.1:3000`. The generated `Home` component renders `Hello from inertia-axum`.
  </Step>
</Steps>

## How it works

`page!` sets the component name to `Home` and serializes `greeting` into the page props. `InertiaApp::vite` reads `frontend/dist/.vite/manifest.json` during startup, generates the browser asset tags, and mounts the build directory at `/build`. `.inertia(inertia)` installs the request and response layer on the router.

The built-in root document handles the first HTML response. When the frontend makes an Inertia visit, the same route returns a JSON page object instead of rebuilding the document.

## Defaults and edge cases

- The generated Vite entry is `src/main.ts`, matching the adapter default.
- The component name is case-sensitive and must resolve to `frontend/src/Pages/Home.*`.
- `build()` reads the production manifest immediately. Run the frontend build first or set `VITE_DEV_SERVER_URL` for a live Vite server.
- A custom root template is not required.
- This is CSR startup. If you configure SSR, replace `build()` with `start().await`.

## Production guidance

Commit `frontend/pnpm-lock.yaml`, rebuild the frontend for every deployment, and start the Rust binary only after the manifest exists. Use the framework-specific guide to adopt the tested `@inertiajs/vite` configuration and the SSR guide before enabling server rendering.
