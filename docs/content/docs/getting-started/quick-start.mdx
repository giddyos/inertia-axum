Build a first page that renders a greeting immediately, shows a loading fallback, and then reveals stats produced by slower server work.

<Steps>
  <Step>
    ### Create the project

    ```bash title="Terminal"
    cargo new inertia-demo
    cd inertia-demo
    ```
  </Step>
  <Step>
    ### Add the dependencies

    ```bash title="Terminal"
    cargo add axum@0.8.9 --features http1,tokio
    cargo add inertia-axum@=1.0.0-alpha.1
    cargo add serde@1 --features derive
    cargo add tokio@1 --features macros,net,rt-multi-thread,time
    cargo install cargo-inertia --version 1.0.0-alpha.1 --locked
    ```
  </Step>
  <Step>
    ### Generate one frontend

    <Tabs items={['Svelte', 'React', 'Vue']}>
      <Tab value="Svelte">
        ```bash title="Terminal"
        cargo inertia init --frontend svelte
        ```
      </Tab>
      <Tab value="React">
        ```bash title="Terminal"
        cargo inertia init --frontend react
        ```
      </Tab>
      <Tab value="Vue">
        ```bash title="Terminal"
        cargo inertia init --frontend vue
        ```
      </Tab>
    </Tabs>
  </Step>
  <Step>
    ### Add the Rust server

    Replace `src/main.rs` with the checked quick-start server:

    <Snippet language="rust" source="quick-start-server/src/main.rs" title="src/main.rs" />
  </Step>
  <Step>
    ### Preview the generated page

    The selected `Home` component uses its adapter's `Deferred` boundary:

    <Tabs items={['Svelte', 'React', 'Vue']}>
      <Tab value="Svelte">
        <Snippet language="svelte" source="svelte/src/Pages/Home.svelte" title="frontend/src/Pages/Home.svelte" />
      </Tab>
      <Tab value="React">
        <Snippet language="tsx" source="react/src/Pages/Home.tsx" title="frontend/src/Pages/Home.tsx" />
      </Tab>
      <Tab value="Vue">
        <Snippet language="vue" source="vue/src/Pages/Home.vue" title="frontend/src/Pages/Home.vue" />
      </Tab>
    </Tabs>
  </Step>
  <Step>
    ### Install and build

    ```bash title="Terminal"
    pnpm --dir frontend install
    pnpm --dir frontend build
    ```
  </Step>
  <Step>
    ### Run the application

    ```bash title="Terminal"
    cargo run
    ```

    Open `http://127.0.0.1:3000`.
  </Step>
</Steps>

`greeting` arrives in the initial response. The simulated slow `load_stats` work is deferred so it cannot hold up that response; the client shows `Loading stats…`, requests `stats`, and replaces the fallback with `3 projects · 12 tasks` when the follow-up finishes.
