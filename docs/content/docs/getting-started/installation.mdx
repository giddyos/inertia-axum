## Requirements

Install Rust 1.88 or newer, Node.js 22.12 or newer, and pnpm 11. The Node requirement comes from the Vite-based maintained examples.

```toml title="Cargo.toml"
[dependencies]
axum = { version = "0.8.9", features = ["http1", "tokio"] }
inertia-axum = "=1.0.0-alpha.1"
serde = { version = "1", features = ["derive"] }
tokio = { version = "1", features = ["macros", "net", "rt-multi-thread", "time"] }
```

The `time` feature supports the quick start's simulated slow query. Install the optional scaffold CLI if you want it to generate the frontend files:

```bash title="Terminal"
cargo install cargo-inertia --version 1.0.0-alpha.1 --locked
```

For Actix Web, use the adapter with the same shared runtime and feature names:

```toml title="Cargo.toml"
[dependencies]
actix-web = "4.14.0"
inertia-actix = "=1.0.0-alpha.1"
serde = { version = "1", features = ["derive"] }
serde_json = "1"
```

Continue with [Actix Web setup](/docs/getting-started/actix-web), or use the
Axum quick start below.

For Rocket, attach the adapter fairing:

```toml title="Cargo.toml"
[dependencies]
inertia-rocket = "=1.0.0-alpha.1"
rocket = "0.5.1"
serde = { version = "1", features = ["derive"] }
```

Continue with [Rocket setup](/docs/getting-started/rocket).

| Default feature | Purpose |
| --- | --- |
| `macros` | Re-exports the typed page, prop, and form derives. |
| `vite` | Reads the Vite manifest and serves built frontend assets. |

See [Cargo features](/docs/reference/cargo-features) for SSR, validation, transient storage, templates, testing integrations, and no-default-feature setups. Then build the [Axum quick-start application](/docs/getting-started/quick-start), its [Actix Web equivalent](/docs/getting-started/actix-web), or the [Rocket setup](/docs/getting-started/rocket).
