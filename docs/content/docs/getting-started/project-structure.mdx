Keep Axum routes and frontend pages in one project so their component names remain easy to trace.

## Conventional layout

<Files>
  <File name="Cargo.toml" />
  <Folder name="src" defaultOpen>
    <File name="main.rs" />
  </Folder>
  <Folder name="frontend" defaultOpen>
    <File name="package.json" />
    <File name="pnpm-lock.yaml" />
    <File name="vite.config.ts" />
    <Folder name="src" defaultOpen>
      <File name="main.ts" />
      <Folder name="Pages" defaultOpen>
        <File name="Home.svelte (or Home.tsx / Home.vue)" />
        <Folder name="Users">
          <File name="Show.svelte (or Show.tsx / Show.vue)" />
        </Folder>
      </Folder>
    </Folder>
    <Folder name="dist">
      <Folder name=".vite">
        <File name="manifest.json" />
      </Folder>
    </Folder>
  </Folder>
</Files>

The Rust component name selects a file below `frontend/src/Pages`:

```rust title="src/routes/users.rs"
async fn show_user() -> DynamicPage {
    page!("Users/Show", { name: "Ada" })
}
```

`Users/Show` resolves to `Pages/Users/Show.svelte`, `Show.tsx`, or `Show.vue`. Vite records `src/main.ts` and its hashed browser assets in `frontend/dist/.vite/manifest.json`.

Next, use [typed pages and props](/docs/guides/typed-pages-and-props) to replace string component names where compile-time contracts help.
