Connect to a separately deployed SSR service when Node needs its own container,
cluster, release cycle, or process manager. Rust waits for readiness and sends
render requests, but it does not start, restart, or stop the external process.

## Prerequisites

- Enable the `ssr` feature.
- Run an Inertia-compatible service with `GET /health` and `POST /render`.
- Expose it through an absolute `http://` URL reachable from the Axum process.
- Build the browser client manifest when using `InertiaApp::vite(...)`.

## Configure the endpoint

```rust
use inertia_axum::{InertiaApp, Ssr};

let inertia = InertiaApp::vite("frontend")
    .ssr(Ssr::external("http://ssr.internal:13714"))
    .start()
    .await?;
```

Startup polls `/health` until it succeeds or `startup_timeout` expires. Page
renders are sent as JSON to `/render`. The response must be `null` or an object
with a string `body` and an array of string `head` fragments.

The adapter accepts plain HTTP endpoints only. Keep the service on a private
network or loopback interface. If TLS is required at a network boundary,
terminate it at a proxy and connect Rust to the trusted internal HTTP endpoint.

## Choose bundle validation

External mode does not check for a local bundle by default. This is the normal
choice when Rust and Node use different images or filesystems.

Add a local check when both processes share the build output:

```rust
let ssr = Ssr::external("http://127.0.0.1:13714")
    .bundle("dist/ssr/app.js");
```

Calling `.skip_bundle_check()` after `.bundle(...)` disables that check again.
With a Vite root, a relative validation path is resolved from that root.

## Define ownership clearly

The external supervisor owns:

- process startup, restarts, scaling, and shutdown;
- Node logs and metrics;
- network policy and service discovery;
- cluster capacity and rolling deployments.

The Rust adapter owns render timeouts, response-size validation, load shedding,
fallback or strict behavior, and local health classification. It does not
background-probe an external service after startup; later failures are recorded
when a render is attempted.

Set Rust's `max_concurrency` from the measured capacity of the whole external
cluster. Continue with [Fallback and strict mode](/docs/ssr/fallback-and-strict-mode)
and [Health and operations](/docs/ssr/health-and-operations).
