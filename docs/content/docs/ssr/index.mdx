SSR adds rendered markup to eligible initial page visits without changing Axum handlers or later Inertia JSON visits.

## Enable SSR

<Steps>
  <Step>
    ### Enable the feature

    ```toml title="Cargo.toml"
    [dependencies]
    inertia-axum = { version = "=1.0.0-alpha.1", features = ["ssr"] }
    ```
  </Step>
  <Step>
    ### Build client and SSR outputs

    ```json title="frontend/package.json"
    {
      "scripts": {
        "build": "vite build && vite build --ssr"
      }
    }
    ```

    Use the framework configurations and output contract in [Frontend build](/docs/ssr/frontend-build).
  </Step>
  <Step>
    ### Start asynchronously

    ```rust title="src/main.rs"
    let inertia = InertiaApp::vite("frontend")
        .ssr("dist/ssr/app.js")
        .start()
        .await?;
    ```
  </Step>
</Steps>

Configured SSR is enabled by default for initial page `GET` requests unless a route policy opts out. Render failures fall back to the normal usable CSR document by default; select [strict mode](/docs/ssr/fallback-and-strict-mode) only when SSR is a hard requirement.

[Managed Node](/docs/ssr/managed-node) makes Rust supervise one renderer process. [External SSR](/docs/ssr/external-server) connects Rust to a service whose supervisor owns process lifecycle and scaling.
