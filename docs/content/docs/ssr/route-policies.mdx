Route policies affect only eligible initial page `GET` requests. Inertia JSON visits, writes, and ordinary Axum responses never invoke SSR.

## Default-on

Configured SSR needs no route marker:

```rust title="src/router.rs"
use axum::routing::get;

let dashboard = get(dashboard);
```

## Opt a route out

```rust title="src/router.rs"
use inertia_axum::SsrRouteExt as _;

let account = get(account).without_ssr();
```

`.without_ssr()` is the prominent choice for authenticated or browser-only pages that should always return CSR.

## Use explicit opt-in

```rust title="src/router.rs"
use inertia_axum::{Ssr, SsrRouteExt as _};

let ssr = Ssr::node("dist/ssr/app.js").opt_in();
let marketing = get(marketing).with_ssr();
```

Unmarked routes remain CSR in opt-in mode.

## Select SSR per request

```rust title="src/router.rs"
use inertia_axum::SsrRouteExt as _;

let preview = get(preview).ssr_when(|context| {
    !context.headers().contains_key("x-force-csr")
});
```

`ssr_when` returns an explicit decision and can inspect the method, URI, headers, or request extensions. Put asynchronous decisions in middleware, store the small result in an extension, and keep the condition synchronous.

The policy nearest the handler wins: method-route policy, then nested router policy, then outer router policy. Test every branch without Node through [SSR testing](/docs/ssr/testing).
