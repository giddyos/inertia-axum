## What you will accomplish

You will choose a supported Inertia v3 client and connect its page resolver to component names returned by Axum.

| Framework | Supported generation | Setup guide | Maintained example |
| --- | --- | --- | --- |
| Svelte | Svelte 5 | [Configure Svelte](/docs/frontend/svelte) | `examples/axum-svelte` |
| React | React 19 | [Configure React](/docs/frontend/react) | `examples/axum-react` |
| Vue | Vue 3 | [Configure Vue](/docs/frontend/vue) | `examples/axum-vue` |

## Prerequisites and features

Complete [Frontend setup](/docs/getting-started/frontend-setup). Every framework uses the default `vite` Cargo feature. Add `ssr` only when following the SSR section.

## Minimal shared implementation

All three clients consume the same page response:

```rust
async fn index() -> DynamicPage {
    page!("Home", { greeting: "Hello" })
}
```

Choose the matching page file:

## How it works

The server does not render framework components during an ordinary CSR request. It returns a component name and serialized props. The selected Inertia client resolves the component, mounts it for the first response, and swaps it during later Inertia visits. The Rust routing and page contract remain the same when you change frontend framework.

## Defaults and edge cases

- The built-in root uses the `app` mount id expected by the clients.
- Component names and filesystem case must match.
- `@inertiajs/vite` can generate the default resolver for conventional `Pages`/`pages` directories; the Svelte example uses an explicit eager resolver.
- Client-side prop types do not replace server serialization or validation.
- SSR is optional. CSR remains usable when no SSR backend is configured or when a configured backend falls back.

## Production guidance

Pin frontend dependencies through `pnpm-lock.yaml`, run a clean client build in CI, and verify the configured manifest entry exists before shipping the Rust binary. When enabling SSR, build both client and SSR outputs and keep framework hydration behavior covered by browser tests.
