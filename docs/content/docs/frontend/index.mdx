Svelte, React, and Vue consume the same component name and props. The quick-start files are `Home.svelte`, `Home.tsx`, and `Home.vue`.

```rust title="src/routes/home.rs"
page!("Home", {
    greeting: "Hello",
    stats: defer(load_stats),
})
```

## Deferred boundary comparison

<Tabs items={['Svelte', 'React', 'Vue']}>
  <Tab value="Svelte">
    ```svelte title="frontend/src/Pages/Home.svelte"
    <Deferred data="stats">
      {#snippet fallback()}<p>Loading stats…</p>{/snippet}
      <p>{stats.projects} projects · {stats.tasks} tasks</p>
    </Deferred>
    ```
  </Tab>
  <Tab value="React">
    ```tsx title="frontend/src/Pages/Home.tsx"
    <Deferred data="stats" fallback={<p>Loading stats…</p>}>
      <p>{stats.projects} projects · {stats.tasks} tasks</p>
    </Deferred>
    ```
  </Tab>
  <Tab value="Vue">
    ```vue title="frontend/src/Pages/Home.vue"
    <Deferred data="stats">
      <template #fallback><p>Loading stats…</p></template>
      <p>{{ stats.projects }} projects · {{ stats.tasks }} tasks</p>
    </Deferred>
    ```
  </Tab>
</Tabs>

Choose [Svelte](/docs/frontend/svelte), [React](/docs/frontend/react), or [Vue](/docs/frontend/vue) for its complete client setup. All three continue with the same [deferred-data workflow](/docs/guides/deferred-and-lazy-props).
