The CLI scaffold is a CSR-focused TypeScript setup using `src/main.ts` and `Home.tsx`. The maintained production example intentionally uses JavaScript/JSX, `@inertiajs/vite`, a custom `src/app.jsx` entry, client and SSR builds, and browser hydration coverage.

## Configure React

<Steps>
  <Step>
    ### Install the tested packages

    ```bash title="Terminal"
    pnpm --dir frontend add @inertiajs/react@3.6.1 react@19.2.4 react-dom@19.2.4
    pnpm --dir frontend add -D @inertiajs/vite@3.6.1 @vitejs/plugin-react@6.0.0 vite@8.1.4
    ```
  </Step>
  <Step>
    ### Configure Vite

    ```js title="frontend/vite.config.js"
    import inertia from '@inertiajs/vite'
    import react from '@vitejs/plugin-react'
    import { defineConfig } from 'vite'

    export default defineConfig({
      plugins: [react(), inertia()],
      build: {
        manifest: true,
        rolldownOptions: {
          input: 'src/app.jsx',
        },
      },
    })
    ```
  </Step>
  <Step>
    ### Boot Inertia

    ```jsx title="frontend/src/app.jsx"
    import { createInertiaApp } from '@inertiajs/react'

    createInertiaApp({ strictMode: true })
    ```
  </Step>
  <Step>
    ### Render the shared Home page

    <Snippet language="tsx" source="react/src/Pages/Home.tsx" title="frontend/src/Pages/Home.tsx" />

    The matching server contract is in the [quick start](/docs/getting-started/quick-start).
  </Step>
  <Step>
    ### Build

    ```bash title="Terminal"
    pnpm --dir frontend build
    ```
  </Step>
  <Step>
    ### Continue to production rendering

    Compare the intentional JSX in [`axum-react`](https://github.com/giddyos/inertia-axum/tree/main/examples/axum-react) and add its client/SSR entries through the [SSR frontend build](/docs/ssr/frontend-build).
  </Step>
</Steps>
