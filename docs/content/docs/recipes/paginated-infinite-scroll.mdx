Return a `ScrollPage` so the Inertia client can load adjacent pages, merge activities, and synchronize the visible page number.

```rust title="src/routes/activity.rs"
use axum::extract::Query;
use inertia_axum::{ScrollPage, prelude::*};

async fn index(Query(query): Query<Pagination>) -> DynamicPage {
    let result = load_page(query.page).await;
    let mut activities = ScrollPage::new(result.items, query.page)
        .page_name("page");

    if query.page > 1 {
        activities = activities.previous(query.page - 1);
    }
    if result.has_more {
        activities = activities.next(query.page + 1);
    }

    page!("Activity/Index", {
        activity: scroll(activities).match_on("id"),
    })
}
```

    import { InfiniteScroll } from '@inertiajs/react'

      return (
        <InfiniteScroll data="activity">
          {activity.data.map((item) => (

          ))}
        </InfiniteScroll>

## Reset after filtering

```ts title="frontend/src/Pages/Activity/filters.ts"
router.reload({
  data: { status },
  only: ['activity'],
  reset: ['activity'],
})
```

Give independent scroll regions different `.page_name(...)` values. Exact merge, reset, and pagination metadata remains in [Props and policies](/docs/reference/props-and-policies).
