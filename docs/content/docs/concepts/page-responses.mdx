---
title: Page responses
description: Return concise dynamic pages or compiler-checked typed pages from Axum handlers.
---

An Inertia handler returns a component name and its props. Use `page!` for small
or dynamic responses; derive `InertiaPage` when a page has a stable contract used
by handlers and tests.

## Return a dynamic page

The unconditional `page!` macro accepts ordinary serializable values and composable
`Prop<T>` values. It does not require the `macros` feature.

```rust
use inertia_axum::prelude::*;

async fn index() -> DynamicPage {
    page!("Home", {
        app_name: "Acme",
        message: "Rendered by Axum through Inertia.",
    })
}
```

Use the builder when keys or page flags are conditional:

```rust
use axum::http::StatusCode;
use inertia_axum::DynamicPage;

let page = DynamicPage::new("Reports/Show")
    .prop("reportId", 42_u64)
    .status(StatusCode::CREATED)
    .encrypt_history()
    .preserve_fragment();
```

`clear_history()`, `encrypt_history()`, and `preserve_fragment()` set Inertia v3
page metadata. `flash(key, value)` writes outside `props` and browser history; it
requires a configured transient store.

## Derive a typed page

Enable the `macros` feature (enabled by default) and derive `InertiaPage` on a
named-field struct:

```rust
use inertia_axum::prelude::*;
use serde::Serialize;

#[derive(Serialize)]
struct Todo {
    id: u64,
    title: String,
}

#[derive(InertiaPage)]
#[inertia(component = "Todos/Index", rename_all = "camelCase")]
struct TodosPage {
    todos: Vec<Todo>,
    can_create: bool,
}

async fn index() -> TodosPage {
    TodosPage {
        todos: vec![Todo { id: 1, title: "Ship docs".into() }],
        can_create: true,
    }
}
```

The derive generates `TodosPage::COMPONENT` and typed top-level keys such as
`TodosPage::TODOS` and `TodosPage::CAN_CREATE`. `inertia-axum-test` uses those keys
for compile-checked selection and assertions.

Fields support `#[inertia(rename = "...")]` and `#[inertia(skip)]`. Page and props
derives support `camelCase`, `snake_case`, `kebab-case`, and `PascalCase` rename
rules. The serialized top-level key `errors` is reserved for validation.

Continue to [Typed pages and props](/docs/guides/typed-pages-and-props) or see the
exact [macro reference](/docs/reference/macros).
