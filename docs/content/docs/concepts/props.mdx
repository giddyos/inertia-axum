Props are serializable values with a loading policy that decides when the server resolves them.

## Loading policies

| Policy | Initial visit | Matching partial request |
| --- | --- | --- |
| Immediate | Included | Included when selected |
| `always` | Included | Included regardless of selection |
| `lazy` | Included | Included when selected |
| `optional` | Omitted | Included only when selected |
| `defer` | Advertised, then omitted | Loaded by its follow-up group |

```rust title="src/routes/dashboard.rs"
use inertia_axum::prelude::*;

async fn dashboard() -> DynamicPage {
    page!("Dashboard", {
        title: "Overview",
        audit: optional(load_audit),
        stats: defer(load_stats).group("summary"),
    })
}
```

Loading, rescue, grouping, merge, scroll, and once policies compose through builder methods. Use [Deferred and lazy props](/docs/guides/deferred-and-lazy-props), [Partial reloads](/docs/guides/partial-reloads), and [Merge, scroll, once, and reset](/docs/guides/merge-scroll-once-and-reset) for workflows; [Props and policies](/docs/reference/props-and-policies) retains exact selection rules, metadata, precedence, and limits.
