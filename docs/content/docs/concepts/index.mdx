These concepts explain the request lifecycle and the adapter abstractions you will
use in application code. They are organized around building an application, not
around crate modules.

## Learn in order

1. [How Inertia works](/docs/concepts/how-inertia-works) — initial HTML, later JSON,
   and the page object.
2. [Application and router](/docs/concepts/application-and-router) — configure one
   `InertiaApp` and install it on Axum.
3. [Page responses](/docs/concepts/page-responses) — return dynamic or typed pages.
4. [Props](/docs/concepts/props) — choose when data is loaded and how it is applied.
5. [Shared data](/docs/concepts/shared-data) — prepare application-wide props from
   each request.

For copyable procedures, continue to the [guides](/docs/guides/vite-and-assets).
For exact APIs and defaults, use the [reference](/docs/reference).
