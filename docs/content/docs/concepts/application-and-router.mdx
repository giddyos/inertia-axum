---
title: Application and router
description: Build one Inertia application and install it around the complete Axum router.
---

Build the immutable `InertiaApp` once at startup, then install its layer after composing routes and state.

## Client-side rendering

```rust title="src/main.rs"
use axum::{Router, routing::get};
use inertia_axum::prelude::*;

let inertia = InertiaApp::vite("frontend").build()?;

let app = Router::new()
    .route("/", get(|| async { page!("Home", {}) }))
    .route("/health", get(|| async { "ok" }))
    .inertia(inertia);
```

`.inertia(...)` belongs on the outer router so request parsing and response finalization wrap every page route. Returning a page without that layer produces a diagnostic 500.

## Server-side rendering

```rust title="src/main.rs"
let inertia = InertiaApp::vite("frontend")
    .ssr("dist/ssr/app.js")
    .start()
    .await?;
```

Use synchronous `build()` for CSR and asynchronous `start().await` when an SSR runtime is configured. See [Configuration](/docs/reference/configuration) for builder defaults and replacement precedence, and [SSR](/docs/ssr) for runtime ownership.
