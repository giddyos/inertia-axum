`inertia-axum` is an Inertia.js v3 server adapter for Axum. Keep routes, state,
extractors, middleware, validation, and data loading in Rust while rendering pages
with Svelte, React, or Vue—without maintaining a separate JSON API.

## What you can build

- Dynamic `page!` responses or strongly typed pages and props.
- Shared, request-aware, lazy, optional, deferred, rescued, mergeable, scroll,
  and once-only data.
- Redirect-based form validation with error bags, old input, and flash values.
- Built-in, marker-based, Askama, or fully custom root documents.
- Client-side rendering and server-side rendering from the same Axum routes.
- In-process application and SSR tests with `inertia-axum-test`.

## Start here

New to the adapter? Follow [Getting started](/docs/getting-started). Already have

releases may change before 1.0; pin the version and review migration notes when
upgrading.
