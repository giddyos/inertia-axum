`inertia-axum` is an Inertia.js v3 server adapter for Axum. Keep routing, data loading, validation, and redirects in Rust while Svelte, React, or Vue renders the pages—without a separate JSON API.

The framework-neutral runtime also powers the first-party
[`inertia-actix` adapter](/docs/getting-started/actix-web) and
[`inertia-rocket` adapter](/docs/getting-started/rocket), with the same page,
prop, form, asset, transient, and SSR behavior.

## Choose an example

- [`axum-svelte`](https://github.com/giddyos/inertia-axum/tree/main/examples/axum-svelte), [`axum-react`](https://github.com/giddyos/inertia-axum/tree/main/examples/axum-react), and [`axum-vue`](https://github.com/giddyos/inertia-axum/tree/main/examples/axum-vue) show maintained framework applications.

- [`incident-board`](https://github.com/giddyos/inertia-axum/tree/main/examples/incident-board) shows advanced prop policies.

This documentation targets `1.0.0-alpha.1` and Rust 1.88. Pin the exact prerelease version and review the [migration guide](/docs/migrations/from-0-5) when upgrading from 0.5.
