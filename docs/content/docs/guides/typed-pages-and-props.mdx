---
title: Typed pages and props
description: Derive compiler-checked page contracts and typed prop keys for handlers and tests.
---

Use `InertiaPage` for a stable page response and `InertiaProps` for reusable typed
prop collections. Both derives require the default `macros` feature.

## Prerequisites

```toml
[dependencies]
inertia-axum = "1.0.0-alpha.1"
serde = { version = "1", features = ["derive"] }
```

## Define a page contract

This pattern is reduced from the tested `todo` example:

```rust
use inertia_axum::prelude::*;
use serde::{Deserialize, Serialize};

#[derive(Clone, Serialize, Deserialize)]
pub struct Todo {
    pub id: u64,
    pub title: String,
    pub completed: bool,
}

#[derive(InertiaPage)]
#[inertia(component = "Todos/Index", rename_all = "camelCase")]
pub struct TodosPage {
    pub todos: Vec<Todo>,
    pub stats: Prop<usize>,
    #[inertia(rename = "canCreate")]
    pub can_create: bool,
}
```

Return the struct directly from an Axum handler. Every field is converted separately,
so immediate and asynchronous props can share the same page type.

```rust
use std::convert::Infallible;

async fn index() -> TodosPage {
    TodosPage {
        todos: load_todos().await,
        stats: defer(|| async { Ok::<_, Infallible>(42) }),
        can_create: true,
    }
}
# async fn load_todos() -> Vec<Todo> { vec![] }
```

## Use generated keys

The derive creates:

- `TodosPage::COMPONENT`, a `Component` equal to `Todos/Index`.
- `TodosPage::TODOS`, a `PropKey<Vec<Todo>>`.
- `TodosPage::STATS`, a `PropKey<usize>`.
- `TodosPage::CAN_CREATE`, a `PropKey<bool>` using the serialized name `canCreate`.

These keys keep tests and partial requests aligned with the page definition:

```rust
let page = app
    .inertia_get("/todos")
    .only(TodosPage::STATS)
    .send()
    .await
    .assert_page::<TodosPage>();

let stats: usize = page.prop(TodosPage::STATS);
```

## Derive a prop collection

Use `InertiaProps` when a named struct must convert into the same field-by-field
prop engine without being a page itself. The primary use is a typed `Share` provider.

```rust
#[derive(InertiaProps)]
#[inertia(rename_all = "camelCase")]
struct AppShared {
    app_name: String,
    #[inertia(rename = "auth.user")]
    auth_user: Prop<String>,
}
```

## Attributes and limits

- `#[inertia(component = "...")]` is required and cannot be empty.
- `rename_all` accepts `camelCase`, `snake_case`, `kebab-case`, or `PascalCase`.
- Fields accept `rename = "..."` or `skip`.
- `encrypt_history`, `clear_history`, and `preserve_fragment` are page-level flags.
- Derives support named-field structs, including generics; enums and tuple structs are rejected.
- `errors` is a reserved serialized page field because the adapter manages validation errors.

See [Macros reference](/docs/reference/macros) for the full grammar and
[Testing](/docs/guides/testing) for typed assertions.
