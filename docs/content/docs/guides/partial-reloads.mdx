title: Partial reloads
description: Reload selected props without repeating all data work for a page.

---

Partial reloads let an Inertia visit request only part of the current component's
props. Mark expensive request-only values as `optional`, then select them from the
client with `only` or omit values with `except`.

## Define selectable data

```rust title="src/main.rs"
use inertia_axum::prelude::*;
use std::convert::Infallible;

async fn index() -> DynamicPage {
    page!("Users/Index", {
        users: load_users().await,
        companies: optional(|| async {
            Ok::<_, Infallible>(load_companies().await)
        }),
        request_kind: always("partial-aware"),
    })
}
```

`companies` is absent from a normal visit. It is resolved only when the client
explicitly requests that key. `request_kind` is immediate and remains present even
when a partial request does not select it.

## Request selected props

The call is identical; only the adapter import differs:

<Tabs items={['Svelte', 'React', 'Vue']}>
  <Tab value="Svelte">
    ```ts title="frontend/src/Pages/Users/Index.svelte"
    import { router } from '@inertiajs/svelte'
    router.reload({ only: ['companies'] })
    ```
  </Tab>
  <Tab value="React">
    ```ts title="frontend/src/Pages/Users/Index.tsx"
    import { router } from '@inertiajs/react'
    router.reload({ only: ['companies'] })
    ```
  </Tab>
  <Tab value="Vue">
    ```ts title="frontend/src/Pages/Users/Index.vue"
    import { router } from '@inertiajs/vue3'
    router.reload({ only: ['companies'] })
    ```
  </Tab>
</Tabs>

The client sends `X-Inertia-Partial-Component` and selection headers. No Axum route
or handler changes are required.

Filtering applies to matching Inertia `GET` requests. Exact component matching,
`only`/`except` precedence, always-prop behavior, and selection timing remain in
[Props and policies](/docs/reference/props-and-policies).

## Test the request

```rust title="src/main.rs"
let page = app
    .inertia_get("/users")
    .only(UsersPage::COMPANIES)
    .send()
    .await
    .assert_page::<UsersPage>();

page.assert_missing(UsersPage::USERS);
```

For data loaded automatically after the first render, use
[Deferred and lazy props](/docs/guides/deferred-and-lazy-props). For wire-level
details, see [Request headers](/docs/reference/request-headers).
