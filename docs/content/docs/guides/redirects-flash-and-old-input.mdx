Return `Redirect` for application navigation and `Location` when Inertia must leave
the application. Attach flash values to redirects when the destination page needs a
one-request success message or identifier.

## Redirect inside the application

```rust title="src/main.rs"
use inertia_axum::prelude::*;

async fn store() -> Redirect {
    Redirect::to("/projects").flash("message", "Project created")
}
```

Write methods become `303 See Other`, ensuring the browser follows with `GET`.
Ordinary `GET` redirects use `302 Found`.

Use `Redirect::back()` to prefer the request `Referer` and fall back to `/`, or
`Redirect::back_or("/projects")` for an explicit fallback.

## Read flash on the destination

Flash is emitted as top-level `page.flash`, separate from `props`, and is not kept in
browser history. It is consumed once.

<Tabs items={['Svelte', 'React', 'Vue']}>
  <Tab value="Svelte">
    ```svelte title="frontend/src/Layouts/App.svelte"
    <script>
      import { usePage } from '@inertiajs/svelte'
      const page = usePage()
    </script>

    {#if page.flash?.message}<p>{page.flash.message}</p>{/if}
    ```
  </Tab>
  <Tab value="React">
    ```jsx title="frontend/src/Layouts/App.jsx"
    import { usePage } from '@inertiajs/react'

    const { flash } = usePage()
    return flash?.message && <p>{flash.message}</p>
    ```
  </Tab>
  <Tab value="Vue">
    ```vue title="frontend/src/Layouts/App.vue"
    <script setup>
    import { usePage } from '@inertiajs/vue3'
    const page = usePage()
    </script>

    <template><p v-if="page.flash?.message">{{ page.flash.message }}</p></template>
    ```
  </Tab>
</Tabs>

`DynamicPage::flash(...)` attaches a value only to the current response. It does not
persist it to the following request.

Both redirect flash and validation errors require
`.transient(store)`. If no store is configured, the adapter returns an actionable
`500` instead of silently losing data. Redirects and stale-version conflicts reflash
unconsumed values so they survive to the eventual page.

## Validation errors and old input

`Validated<T>` failures use the same one-request transport. The next page receives:

- `props.errors`, optionally nested beneath an error bag.
- `props.oldInput` only when the form derive enabled `old_input`.

Sensitive or redacted fields are omitted before storage. See
[Forms and validation](/docs/guides/forms-and-validation).

## Leave the Inertia application

```rust title="src/main.rs"
use inertia_axum::Location;

async fn billing() -> Location {
    Location::external("https://billing.example.com/portal")
}
```

The Inertia client performs a full browser visit for an external location. See
[Protocol behavior](/docs/advanced/protocol-behavior) for status and header details, and
[Transient storage](/docs/guides/transient-storage) for production stores.
