Defer slow dashboard work so the page shell and immediate title reach the browser first.

```rust title="src/routes/dashboard.rs"
async fn dashboard() -> DynamicPage {
    page!("Dashboard", {
        title: "Overview",
        stats: defer(load_stats),
        activity: defer(load_activity)
            .group("activity")
            .rescue(),
    })
}
```

## Grouping

Deferred props use the `default` group unless `.group("activity")` assigns another one. Each group produces its own follow-up request, so align groups with UI regions that can reveal together.

## Rescue and retry

`.rescue()` keeps a failed prop from turning the response into a generic 500 and adds its key to `rescuedProps`. Each adapter's rescue slot receives `reloading`; `router.reload({ only: ['activity'] })` retries only that prop.

| Helper | Use it when |
| --- | --- |
| `lazy` | Data belongs on the initial visit but should resolve only when selected. |
| `optional` | Data should resolve only after an explicit partial reload. |
| `defer` | Data should load automatically after the initial render. |

See [Props and policies](/docs/reference/props-and-policies) for exact selection, concurrency, and failure semantics.
