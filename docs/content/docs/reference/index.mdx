## Public API groups

The main crate deliberately exposes a small common path and several advanced extension points:

- Application setup: `InertiaApp`, `InertiaAppBuilder`, `RouterInertiaExt`, `InertiaLayer`, and `InertiaService`.
- Direct responses: `DynamicPage`, `page!`, `Redirect`, `Location`, and typed structs derived with `InertiaPage`.
- Props: `Prop<T>`, the helper functions, `ScrollPage<T>`, prop policy models, and generated `PropKey<T>` constants.
- Request-aware data: `Share`, `ShareContext`, `Visit`, and `RequestContext`.
- Roots and assets: `RootView`, `RootContext`, trusted markup wrappers, `AssetProvider`, and `AssetVersion`.
- Forms and transient data: `Validated<T>`, `Validate`, `Errors`, `TransientStore`, and the built-in stores.
- Optional SSR: `Ssr`, route policies, health models, and classified start/render failures.

The `inertia_axum::prelude` module contains the usual handler and setup imports. `inertia_axum::advanced` collects extension interfaces. `inertia_axum::compat` retains the explicit rendering APIs used while migrating from 0.5. See [compatibility APIs](/docs/advanced/compatibility-apis) before starting new code with `Inertia<T>` or `InertiaRequest`.

## Version and stability

This documentation describes `1.0.0-alpha.1`. The workspace MSRV is Rust 1.88. Alpha releases may still change public APIs; pin the exact version when reproducible builds matter.
